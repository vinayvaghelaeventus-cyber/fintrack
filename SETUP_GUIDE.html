🔥 FinTrack v4 — Setup Guide

✓ Free ✦ Firebase Cloud Storage ✦ PIN Lock ✦ Auto Salary ✦ CC EMI Tracking

1
Get your Firebase Config (3 minutes)

1. Go to console.firebase.google.com

2. Click your existing project (or Add project if needed)

3. Click the ⚙️ gear icon → Project Settings

4. Scroll to "Your apps" → click Web (the </> icon)

5. Register with nickname fintrack → click Register app

6. Copy the firebaseConfig object — it looks like this:

const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "yourproject.firebaseapp.com",
  projectId: "yourproject-id",
  storageBucket: "yourproject.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};
💡 Keep this config private — don't share it publicly. Your PIN + Firestore rules protect your data.
2
Enable Firestore Database

1. In Firebase Console sidebar → click Firestore Database

2. Click Create database

3. Choose Start in test mode (we'll secure it next) → Next

4. Select a region (e.g. asia-south1 for India) → Enable

✅ Firestore is now active. Your data will sync here from the app.
3
Set Firestore Security Rules (important!)

In Firestore Console → click Rules tab → replace the content with:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /fintrack_users/{userId} {
      allow read, write: if true;
    }
  }
}

Click Publish.

💡 This allows your app to read/write its own document. Your PIN provides the user-level security. For extra security, come back later and restrict by IP or add Firebase Auth.
4
Paste your config into the app

Open src/firebaseConfig.js and replace the placeholder values:

export const firebaseConfig = {
  apiKey:            "YOUR_apiKey_HERE",
  authDomain:        "YOUR_authDomain_HERE",
  projectId:         "YOUR_projectId_HERE",
  storageBucket:     "YOUR_storageBucket_HERE",
  messagingSenderId: "YOUR_messagingSenderId_HERE",
  appId:             "YOUR_appId_HERE",
};

// Change this to something unique (e.g. your name)
export const FINTRACK_USER_ID = "rahul_fintrack_2024";
💡 Change FINTRACK_USER_ID to something personal and unique. This is your data's document path in Firestore — only someone who knows this exact string can access your data.
5
Upload to GitHub & Deploy on Vercel

1. Go to your GitHub repo → delete old files → upload entire fintrack-v4 folder contents

2. Or if you know git: git add . && git commit -m "v4" && git push

3. Vercel will auto-redeploy in ~60 seconds when it detects the push

✅ Your app is live with Firebase. Data now survives app updates, incognito mode, device changes — everything.
6
First launch — Set your PIN

When you open the app for the first time:

1. You'll see a PIN setup screen — enter a 4+ digit PIN

2. Confirm it → tap Set PIN & Open App

3. The PIN is stored in your browser — next time you open the app, you'll need to enter it

💡 The PIN is stored locally (not in Firebase). If you clear browser data, you'll need to set a new PIN — but your financial data is always safe in Firebase.
✦
New Features in v4

🔐 PIN Lock — App locks every time the browser session ends. Keypad entry on mobile.

☁️ Firebase Sync — Auto-saves to cloud every ~1 second. Works across devices, survives updates and incognito.

💰 Auto Salary — Go to ⚙️ Settings → enable Auto Salary → set amount + credit day. Appears automatically every month.

💳 CC EMI Tracking — When adding a credit card, toggle "Has CC EMI?" to track card EMIs separately from loan EMIs.

✏️ Edit Transactions — Tap the ✏️ icon on any transaction to edit it.

🤖 Smart Strategy — App analyses your debts and recommends Avalanche vs Snowball automatically.

📱 PWA Icon — When you add to home screen, FinTrack shows with the ₹ icon.

✦
Future Updates — Your data is always safe

Since data is in Firebase, updating the app code will NEVER affect your data.

To update: just replace src/App.jsx on GitHub. Vercel redeploys automatically. Firebase data stays untouched.

✅ You can update the app as many times as you want. Your transactions, loans, CC data — all safe in Firebase forever.
